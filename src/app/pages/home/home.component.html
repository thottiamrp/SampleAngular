Benefits of UEF 3.0 Standards
  • Standardized interactions with User Interface elements
  • Usability-tested patterns
  • Baked-in accessibility (508 / WCAG 2.0), reviewed by ASB
  • Built in responsiveness to different screen sizes
  • Ensures consistent SSA branding and look and feel
  • Global styles
  • Reusable patterns
 Compatibility
  • Comprised of standards-based web components that run natively in the browser.
  • Framework-agnostic: work in most frameworks and web applications.
  • Provides stability by avoiding framework churn.
  • Supports modern browsers by compiling code down to ES5 and dynamically loading polyfills when needed.
Performance
  • Lazy Loading: By default, components load asynchronously and can be bundled with related components.
  • Virtual DOM: fast DOM updates without common DOM performance pitfalls.
  • Reactivity: Efficient updates based on property and state changes.
  • High-performance Rendering: asynchronous rendering system, similar to React Fiber. (i.e., cutting edge and fast)
 Integration
  • UEF 3 is made to work with other frameworks, providing:
  • Event emitters
  • Two-way data-binding, reactivity*
 Flexibility
  • Building patterns with building block components results is fewer components and properties to remember, and consistent APIs across similar components.
  • UEF 3.0 components are less opinionated, giving the developer more control.
  • UEF 3.0 implements the Flexbox and CSS Grid APIs, allowing for far greater power and flexibility in layout.
